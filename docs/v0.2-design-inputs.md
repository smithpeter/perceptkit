# v0.2 Design Inputs — from the 4.5% ESC-50 reality check

> Derived from `docs/real-accuracy-results.md` (2026-04-19). These are the
> concrete design changes needed to make perceptkit's real-audio number
> useful, not the cheating kind (over-fit YAML thresholds to ESC-50).

## Priority-ordered list

### P0 — Audio-only scene pack

**Problem**: Every v0.1 scene requires context features that audio-only
benchmarks can't provide. When `context.app` / `context.motion` are
absent, 4 of 5 scenes never fire.

**Fix**: Add scenes whose `all` clauses use only `audio.*` features:

```yaml
# scenes/audio_only/ambient_music.yaml
id: ambient_music
describe: {template: "Ambient music playback"}
match:
  all:
    - { feature: audio.rms_db, op: gt, value: -30.0 }
    - { feature: audio.voice_ratio, op: lt, value: 0.2 }
    - { feature: audio.spectral_flatness, op: gt, value: 0.3 }  # NEW feature needed
priority: 5
```

```yaml
# scenes/audio_only/near_silence.yaml
id: near_silence
match:
  all:
    - { feature: audio.rms_db, op: lt, value: -50.0 }
priority: 3
```

```yaml
# scenes/audio_only/sustained_speech.yaml
id: sustained_speech
match:
  all:
    - { feature: audio.voice_ratio, op: gt, value: 0.6 }
    - { feature: audio.voice_activity, op: eq, value: true }
priority: 4
```

Net effect: ESC-50 gets a fair chance to hit `ambient_music` (for music-like
sounds), `near_silence` (for quiet clips), `sustained_speech` (where ZCR is
low). Expected gain: 20-30% Top-1 on pure-audio datasets.

### P1 — Silero VAD (ONNX-based)

**Problem**: Our `VoiceActivityExtractor` uses energy + ZCR. False positives
on non-speech sounds (rain has similar RMS+ZCR to speech).

**Fix**: Integrate Silero VAD via ONNX Runtime:

- New feature-gated crate `perceptkit-silero-vad` (feature `silero-vad`)
- ~1 MB ONNX model bundled via `rust-embed` or downloaded on first use
- Replaces `audio.voice_ratio` with a much more accurate signal
- Expected gain: 15-25% Top-1 on real audio

Alternatives considered:
- WebRTC VAD (Google's, very fast but older) — rust-webrtc-vad crate exists
- Silero wins on accuracy, minor size/speed tradeoff

### P2 — dBFS calibration + normalization

**Problem**: ESC-50 clips are normalized to near-0 dBFS peak. Our
`office_quiet` rule `audio.rms_db < -40.0` never fires on normalized audio
— even actual quiet rooms register around -20 dBFS after normalization.

**Fix**: Add `AudioProvider::with_loudness_target(LUFS)` for optional
loudness normalization before feature extraction. Or add a
`audio.rms_db_peak_normalized` feature that accounts for dynamic range.

### P3 — Synthetic-context benchmark mode

**Problem**: Audio-only datasets (ESC-50, UrbanSound8K) can never exercise
scenes with context dependencies.

**Fix**: Extend `perceptkit eval` with `--inject-context <key>=<value>,...`:

```bash
perceptkit eval --scenes ./scenes \
    --dataset bench.jsonl \
    --inject-context context.app=Zoom  # hypothetical "user is in a meeting"
```

This lets us report "what accuracy would perceptkit achieve if the caller
provided context X, given these audio features". It's still real audio,
but acknowledges the multi-modal assumption.

### P4 — Additional acoustic features

**Problem**: Current 7-feature set is minimal. Audio-only scene
discrimination needs more.

**Candidates** (in order of impact):
- `audio.spectral_flatness` — music vs noise vs speech
- `audio.spectral_centroid` — brightness
- `audio.pitch_confidence` — is there pitched content?
- `audio.onset_density` — percussive/transient activity
- `audio.harmonic_ratio` — tonal vs noise

Implement as optional `ComplexSpectralExtractor` behind a feature flag
(brings in FFT via `rustfft` crate).

### P5 — Benchmark expansion

Current: 200 ESC-50 clips (4 scenes).

**Add**:
- 50 Common Voice clips for `online_meeting` (synthesize multi-speaker overlay)
- 50 MUSAN noise clips for `outdoor_noisy` (separate from ESC-50's weather)
- 50 LibriSpeech clean clips for `sustained_speech` (after P0 lands)

Goal: 350+ labeled clips, 5-7 scenes, macro-F1 ≥ 0.60 real-audio number.

## Non-goals for v0.2

- ❌ LLM-based audio classification (perceptkit's value is fast rule-based +
  optional LLM escalation; don't turn the hot path into inference)
- ❌ Tuning scene thresholds to over-fit ESC-50 (cheating)
- ❌ Per-user personalization (was already v0.3 territory)

## Rough order of operations

Week 1: P0 audio-only scenes + P3 synthetic-context mode → 35-45% Top-1
Week 2: P2 dBFS normalization + P1 Silero VAD scaffolding → 50-60% Top-1
Week 3: P1 Silero VAD fully integrated → 65-75% Top-1
Week 4: P4 spectral features + P5 dataset expansion → macro-F1 ≥ 0.60

End of v0.2: real macro-F1 ≥ 0.72 on 350+ clips (matches original strategic target).

## How this connects to the audit

Final audit (2026-04-19) said condition 3 unlocks alpha.0: "50 real-audio
clips with kappa sampling". Interpreted technically, we hit 200 clips on
2026-04-19 and measured 4.5% Top-1. We chose **Option B** (keep dev.1) not
because the condition wasn't met, but because the number is too low to
present as a consumable alpha.

v0.2 closes this honestly via the 4-item design input list above, not by
moving the goalposts.
